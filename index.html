INTRODUCTION

Visual question answering and text-to-image generation are rapidly advancing frontiers in multimodal AI, powering applications from accessibility tools and educational technology to cultural heritage documentation and creative content generation. While state-of-the-art vision-language models achieve strong results on standard benchmarks, they remain critically under-evaluated on culturally specific visual content, particularly for Arabic, a language spoken by over 400 million people across a region of remarkable cultural diversity. The challenge runs deeper than linguistic complexity: effective Arabic multimodal understanding must generalise across Modern Standard Arabic and major dialects such as Egyptian and Levantine, while remaining grounded in region-specific landmarks, traditions, attire, and everyday practices that existing Western-centric datasets largely ignore or distort. ImageEval 2026 addresses this gap on two fronts: (1) advancing culturally grounded Arabic visual question answering and hallucination detection through Ayn-VQA, built on the large-scale OASIS dataset spanning 18 MENA countries, and (2) benchmarking the cultural accuracy of text-to-image generation for Qatari and Arab cultural scenes through CRAI-Bench, introducing the first structured evaluation framework for this setting. Together, these tasks aim to foster progress in Arabic NLP and multimodal AI by encouraging researchers to develop systems that are not only linguistically robust, but genuinely culturally aware.

Task 1: Ayn-VQA — Culturally Grounded Arabic Visual Question Answering and Hallucination Detection

Overview: Ayn-VQA is built on OASIS (EverydayMMQA framework), a large-scale multimodal dataset spanning 18 MENA countries with approximately 0.92 million images, 14.8 million QA pairs, and 3.7 million spoken questions in MSA, Egyptian Arabic, and Levantine Arabic. The task is organised into three tracks covering spoken question answering, textual question answering, and image-grounded hallucination detection. Each track is offered across all three Arabic varieties.

Subtask 1A: Spoken MCQ Visual Question Answering

Objective: This track evaluates how well systems handle culturally grounded Arabic visual question answering when the question is delivered as speech. Given an image and a spoken Arabic question about its content, participants must predict the correct answer from multiple-choice options. The track is offered in MSA, Egyptian Arabic, and Levantine Arabic, and tests semantic understanding under the additional challenges of speech recognition and prosodic variation across dialects.

Dataset: This track uses a subset of OASIS (EverydayMMQA framework), a large-scale multimodal dataset of images paired with spoken and textual question-answer instances across 18 MENA countries. The shared task subset contains approximately 50,000 to 100,000 images selected to preserve diversity across language variety, country, and topic. Participants will receive four splits: training, development, a dev-test set without labels for leaderboard-based progress tracking, and a hidden test set for final ranking. Data will be distributed via CodaLab after registration.

Submission Format: Participants should submit a TSV file containing one prediction per line. The file must include the following columns:

  • id
  • label (predicted answer option: A, B, C, or D)

The file should be uploaded to CodaLab. Submissions will be automatically validated. Incomplete submissions will be flagged.

Evaluation: The official metric is accuracy: the proportion of questions for which the system selects the correct answer option. F1-score will also be reported as a secondary metric. Systems are ranked by accuracy on the hidden test set.

Subtask 1B: Textual MCQ Visual Question Answering

Objective: This track evaluates culturally grounded Arabic visual question answering using written text input. Given an image and a written Arabic question about its content, participants must predict the correct answer from multiple-choice options. By removing speech as a variable, this track provides a clean measure of visual grounding and cultural understanding across MSA, Egyptian Arabic, and Levantine Arabic.

Dataset: Same OASIS subset as Track 1, using the textual question modality. Participants receive the same four splits: training, development, dev-test (unlabeled), and hidden test. Data will be distributed via CodaLab after registration.

Submission Format: Participants should submit a TSV file containing one prediction per line. The file must include the following columns:

  • id
  • label (predicted answer option: A, B, C, or D)

The file should be uploaded to CodaLab. Submissions will be automatically validated. Incomplete submissions will be flagged.

Evaluation: The official metric is accuracy. F1-score will also be reported as a secondary metric. Systems are ranked by accuracy on the hidden test set.

Subtask 1C: Image-Grounded Hallucination Detection

Objective: This track evaluates whether systems can distinguish image-supported statements from plausible but visually unsupported alternatives. Each instance presents an image together with one grounded true statement (Q+) and two culturally plausible but visually incorrect counterfactual statements (Q-). Participants must judge each statement as true or false. Distractors are drawn from nearby countries, related traditions, visually similar landmarks, and culturally associated objects, making this a genuinely challenging test of visual grounding rather than cultural pattern matching.

Dataset: A contrastive subset of OASIS where each example is structured as a triplet: one grounded true statement and two plausible but visually unsupported counterfactuals. The same four splits apply: training, development, dev-test (unlabeled), and hidden test. Data will be distributed via CodaLab after registration.

Submission Format: Participants should submit a TSV file containing one prediction per line. The file must include the following columns:

  • id
  • triplet_id
  • label (True or False)

The file should be uploaded to CodaLab. Submissions will be automatically validated. Incomplete submissions will be flagged.

Evaluation: The official metric is Contrastive Instability (CI), which measures whether a system resolves the full triplet consistently rather than individual statements in isolation:

CI = 1 - Nconsistent / Npartial

where Npartial is the number of triplets for which the system gets at least one statement correct, and Nconsistent is the number of those triplets fully and correctly resolved. Lower CI is better. Per-statement accuracy is also reported as a secondary metric.

Task 2: CRAI-Bench — Cultural Accuracy Evaluation for Text-to-Image Generation

Overview: CRAI-Bench evaluates whether AI-generated images faithfully represent Qatari and Arab cultural scenes. The task introduces the Cultural Representation Accuracy Index (CRAI), a five-dimensional scoring framework, and is structured into two complementary subtasks: one targeting automatic cultural accuracy evaluation, and one targeting culturally grounded caption generation. A pilot study of 100 image-caption pairs shows that prompt specificity has a statistically significant effect on cultural accuracy (Friedman χ²(4) = 55.25, p < .001, Kendall's W = 0.69), and that LLM-as-a-judge overestimates cultural accuracy compared with human evaluators by 0.29 points (mean LLM 0.73 vs. human 0.44).

Subtask 2A: Cultural Accuracy Evaluation

Objective: Given a reference image of a Qatari cultural scene, a culturally grounded English image caption, and an AI-generated image produced from that caption, participants must produce a CRAI score in [0,1] across five dimensions. Systems are evaluated against human-annotated gold CRAI scores. This subtask benchmarks LLM-as-a-judge approaches against human cultural annotation in Qatari contexts.

Dataset: The dataset consists of reference image-caption-generated image triples grounded in Qatari culture across five categories: people and traditional attire; objects and natural elements; architecture and built environment; cultural activities and practices; and historical and cultural context. Reference images were sourced from open-access platforms (Pexels, Pixabay, Unsplash) and verified for cultural authenticity by native Qatari cultural consultants. Generated images were produced using gpt-image-1 (1024x1024px). The dataset is split approximately 60/20/20 across training, development, and hidden test sets. Full human and LLM CRAI annotation is provided for training and development. Data will be distributed via CodaLab after registration.

Submission Format: Participants should submit a TSV file containing one scored instance per line. The file must include the following columns:

  • id
  • CRAI_CEA (Cultural Element Accuracy, 0-1)
  • CRAI_CC (Contextual Coherence, 0-1)
  • CRAI_CS (Cultural Specificity, 0-1)
  • CRAI_CI (Cultural Integrity, 0-1)
  • CRAI_HP (Hallucination Penalty, 0-1)
  • CRAI_composite (weighted composite score, 0-1)

The file should be uploaded to CodaLab. Submissions will be automatically validated. Incomplete submissions will be flagged.

Evaluation: Systems are ranked by Spearman correlation and mean absolute error (MAE) against human-annotated gold CRAI scores, at both overall and per-dimension levels. The composite CRAI score is computed as:

CRAI = 0.30·CEA + 0.20·CC + 0.20·CS + 0.20·CI − 0.10·HP

Score bands: ≥0.85 highly accurate · 0.70–0.84 mostly accurate · 0.50–0.69 moderate · 0.30–0.49 weak · <0.30 poor/misleading.

Subtask 2B: Culturally Accurate Caption Generation

Objective: Given a reference image of a Qatari cultural scene, participants must generate a culturally grounded English image caption designed to maximise the cultural accuracy of a downstream text-to-image model's output. This subtask targets prompt engineering and culturally grounded caption quality for Qatari settings.

Dataset: Participants receive the same reference images as Subtask A, without the associated captions or generated images. The task is to produce captions that, when passed through a fixed text-to-image system, yield images that score as highly as possible on CRAI. The same 60/20/20 train/dev/test split applies. Data will be distributed via CodaLab after registration.

Submission Format: Participants should submit a TSV file containing one caption per line. The file must include the following columns:

  • id
  • caption

The file should be uploaded to CodaLab. Submissions will be automatically validated. Incomplete submissions will be flagged.

Evaluation: Submitted captions are passed through a fixed text-to-image system; the resulting images are scored with CRAI. Per-caption scores are averaged across k generations (k will be specified at task launch). All submissions are evaluated within a single fixed time window using one API snapshot to control for non-determinism. Systems are ranked by mean CRAI score. Higher is better.


Guidelines for Participating Teams

  • Participants may choose to participate in one or more subtasks.

  • All participants must register through the official website to receive access to datasets and updates.

  • Upon requesting access to the data, participants must agree to submit a 4-page system description paper detailing their approach, methodology, data usage (if external data is used - specify rules!), and findings.

  • Submissions will be peer-reviewed, and selected papers will be published in the Arabic NLP 2026 Conference Proceedings, indexed in the ACL Anthology.

  • Participants are required to create an OpenReview account for paper submission and review processes.
